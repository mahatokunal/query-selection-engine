# Biomedical Sourcing Agent: Streamlined Architecture & Component Guide

---

# THE COMPLETE INTEGRATED ARCHITECTURE (8 Layers)

**Design Principles:** Maximum recall (accuracy) is paramount. Cost and latency are secondary. No capability from the original 12-layer design is lost — overlapping components are merged, not removed.

**What changed from v1 (12 layers):** Merged overlapping workflows ([9C]+[10B] targeted searches, [7A/B/C] triple LLM calls), added incremental deduplication, and made the loop structure explicit (inner/middle/outer).

**What changed from v2 (UCB bandit):** Replaced UCB bandit query selection with farthest-first traversal on sentence embeddings (validated in demo). Queries are no longer bandit arms — workers are the static arms for budget allocation. Added promise score (`novel_assets / total_results`) with exploit/explore budget split. Marked Layer 6 (Modality × Target Alias) as deterministic cross-product. Added multi-dimensional identity scoring for structure-less assets.

**What changed from demo validation:** (1) Round 1 seeding changed from random/input-based to medoid-per-layer — pick one query per layer centroid, eliminates first-round bias. (2) Added bias detection monitoring: 3 metrics (layer distribution, intra-layer distance, distance-to-nearest-tried) that surface embedding model limitations. (3) Flagged that `all-MiniLM-L6-v2` collapses Layer 3 (CJK translations) — must use multilingual embedding model in production. (4) Embeddings cached once at pool creation, reused across rounds. (5) Layer 4 feedback queries require incremental embedding + cache update.
